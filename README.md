# 🗺️ CivicPlus: Geo-Services Module

[cite_start]**A core module of the CivicPlus Smart City Ecosystem.** [cite: 145, 152]

## 📌 Overview
[cite_start]The **Geo-Services** module is an interactive, real-time mapping engine designed to keep citizens moving efficiently and safely throughout the municipality[cite: 152]. By combining crowdsourced data with dynamic routing, this module transforms how residents interact with urban geography and essential city services.

[cite_start]This project was engineered by **Team NOVA** [cite: 162] as part of the CivicPlus Smart City platform.

## 🚀 Key Features

* [cite_start]**"Zahma" Traffic Alerts:** Real-time, crowdsourced road condition tracking to help citizens avoid urban congestion. [cite: 153, 154, 155]
* [cite_start]**Smart Itineraries:** A dynamic, step-by-step routing engine to guide users to their destinations efficiently. [cite: 156, 157]
* [cite_start]**After-Hours "Garde" Locator:** Dedicated mapping to easily find essential services (pharmacies, clinics, shops) operating during nights and weekends. [cite: 158, 159]
* [cite_start]**Dynamic POI Mapping:** An interactive, centralized map displaying critical Points of Interest and municipal facilities across the city. [cite: 160, 161]

## 🛠️ Technology Stack
* **Backend Core:** PHP & MySQL (Relational database for POIs and static geodata)
* **Real-Time Engine:** Node.js (Handling live crowdsourced "Zahma" alerts)
* **Data Processing:** Python Simulation Server (Simulating live traffic states and routing logic)
* **Frontend UI:** HTML5 / CSS3 with a custom Glassmorphism component framework.
* **Typography:** Ubuntu (Headers) & Roboto (Body text).
